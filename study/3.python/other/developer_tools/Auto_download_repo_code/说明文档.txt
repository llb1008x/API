作用
1.repo_init_env脚本作用是初始化repo 工作区



用法
1.拷贝此脚本到开发同事本地任何位置；
2.运行->按照提示输入repo工作区全路径并回车（默认为当前路径）->根据提示输入项目名称并回车即可
（如果您输入的地址已经做过repo初始化则会提示输入新地址）
3.脚本运行完，请输入ls -a 确认已生成.repo 隐藏文件，则repo工作区初始化成功.


注：
1.当前文件夹必须为非repo初始化过的工作区，如果之前被repo初始化过则此次初始化将失败
2此脚本只在新文件夹初始化为repo工作区时使用一次，后续代码更新或提交不需要再次初始化
3.初始化脚本执行完，运行同步命令则git库代码开始同步 如“repo sync” ；（目前所有组都有多有git库权限故使用此命令同步）
4.如果按模块设置了权限，则同步时请输入“repo sync project1 project2”
